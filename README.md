# Novels Paradise - LightNovel Native (منصة جنة الروايات)

منصة متكاملة لقراءة روايات الويب والخفيفة باللغة العربية، تم تحسينها هندسيًا وتأمينها ضد الثغرات الأمنية الشائعة ودعمها ببنية تحتية إنتاجية قابلة للتوسع.

---

## 🛠️ هيكلية المشروع (Architecture)

يتكون المشروع من جزأين رئيسيين:
1. **الواجهة الخلفية (Backend)**: تطبيق Flask متصل بقاعدة بيانات SQLite أو PostgreSQL مع دعم الكاش عبر Redis وتخزين الملفات السحابي Cloudflare R2.
2. **الواجهة الأمامية (Frontend)**: تطبيق صفحة واحدة (SPA) مبني بـ HTML/CSS/JS الأصلي (Vanilla JS)، يدعم التخصيص الكامل لتجربة القراءة والتخزين المحلي.

---

## 🚀 بنية هندسية جاهزة للإنتاج (Production-Ready Architecture)

تمت ترقية البنية التحتية للمشروع لتتحمل معدلات زيارات عالية وتوفر أداءً فائقاً:

### 1. مرونة قاعدة البيانات (Database Portability)
*   **دعم PostgreSQL و SQLite**: تم تطوير طبقة وسيطة تقوم بترجمة الاستعلامات واختلافات الصيغ (Dialect Mapping) تلقائياً، مثل تحويل علامات الاستفهام `?` لـ `%s` وترجمة أوامر `UPSERT / ON CONFLICT` و `PRAGMA`.
*   **تجميع الاتصالات (Connection Pooling)**: تطبيق نظام إدارة تجميع الاتصالات لـ PostgreSQL لتقليل وقت الربط وتحسين الكفاءة.

### 2. التخزين المؤقت وتجميع المشاهدات (Redis Caching & Buffered Views)
*   **طبقة Redis**: تسريع جلب البيانات للصفحة الرئيسية وتفاصيل الروايات والفصول العامة، مع دعم التراجع التلقائي إلى ذاكرة النظام المؤقتة (Mock Cache) في بيئات التطوير.
*   **التحديث المؤجل للمشاهدات (Buffered View Counting)**: لمنع قفل قاعدة البيانات (Database Locking) عند كثرة الزيارات، يتم تخزين زيادة المشاهدات في Redis مؤقتاً ثم تقوم عملية في الخلفية (Background Daemon Thread) بدمجها وتحديث قاعدة البيانات كل 60 ثانية دفعة واحدة.

### 3. تأمين ورفع الصور والوسائط (Media & CDN Uploads)
*   **تحسين صيغة الصور**: يتم تحويل صور الأغلفة والرسومات المرفوعة تلقائياً إلى صيغة **WebP** المحسنة لضغط حجمها وزيادة سرعة التصفح.
*   **التخزين السحابي Cloudflare R2**: يدعم النظام رفع الصور مباشرة إلى سحابة R2 (عبر بروتوكول S3 متوافق) مع التراجع للمجلد المحلي في حال غياب الإعدادات.

## 🔒 التحسينات الأمنية وإدارة الصلاحيات (Security Hardening & Workflows)

تمت معالجة جميع الملاحظات الأمنية وحالات الاستخدام المعقدة لضمان أمان النظام وصلاحياته:

1. **حماية صلاحيات Admin/VIP من التزوير**:
   * لا يتم الوثوق بـ `role` المرفق مع التوكن مباشرة؛ بل يتم جلب هوية المستخدم والتحقق من صلاحياته الفعالة من قاعدة البيانات مباشرة عند كل طلب حساس.

2. **تأمين حساب المدير الافتراضي**:
   * يتوقف التطبيق عن العمل فوراً (Fail-fast) في وضع الإنتاج إذا تم تشغيله ببيانات المدير الافتراضية لمنع الاختراق.

3. **الحماية من ثغرات Stored XSS**:
   * تطوير فلتر مخصص لـ HTML (`SafeHTMLParser`) يقوم بتنظيف محتوى الفصول والإعلانات من الأكواد الخبيثة قبل حفظها وخلال عرضها (Defense-in-depth).
   * **ترحيل أمني تلقائي**: يتم تطهير السجلات القديمة المخزنة مسبقاً في قاعدة البيانات تلقائياً عند بدء التشغيل.

4. **إدارة سير العمل لفريق العمل (Workspace Workflow)**:
   * **إخفاء المسودات**: الفصول غير المنشورة (Draft, In Review, Scheduled) تكون مخفية بالكامل عن الزوار العامين وتتاح فقط للمدراء وأعضاء الفريق المعينين للرواية.
   * **تخطي قفل VIP**: يستطيع المترجمون والمراجعون المعينون للرواية قراءة الفصول المغلقة لـ VIP دون الحاجة لاشتراك مدفوع لتسهيل عملهم.
   * **الترقية التلقائية**: تتم ترقية رتبة المستخدم تلقائياً إلى رتب العمل المناسبة فور تعيينه في فريق إدارة الرواية.
   * **حماية لوحة التحكم**: تم فرض تحقق صارم في الواجهة الأمامية لمنع الوصول غير المصرح به للوحة العمل والتحكم (`#admin`).

## ⚙️ إعدادات متغيرات البيئة (Environment Variables)

قم بإنشاء ملف `.env` في مجلد `lightnovel-backend` لتهيئة المتغيرات التالية:

| اسم المتغير | الوصف | القيمة الافتراضية |
| :--- | :--- | :--- |
| `LIGHTNOVEL_SECRET_KEY` | المفتاح السري لتشفير التوكنات (JWT) | **مطلوب** (يتوقف التطبيق بدونه) |
| `LIGHTNOVEL_DATABASE_URL` | رابط اتصال PostgreSQL (مثال: `postgresql://user:pass@host:port/db`) | غير محدد (يتراجع تلقائياً لـ SQLite) |
| `LIGHTNOVEL_DATABASE_FILE` | مسار ملف قاعدة بيانات SQLite (في حال عدم استخدام Postgres) | `lightnovel.db` |
| `LIGHTNOVEL_REDIS_URL` | رابط اتصال Redis للكاش (مثال: `redis://localhost:6379`) | غير محدد (يتراجع لكاش الذاكرة الافتراضي) |
| `LIGHTNOVEL_ADMIN_USER` | اسم المستخدم الافتراضي للمدير | **مطلوب للإنتاج** |
| `LIGHTNOVEL_ADMIN_PASSWORD` | كلمة المرور الافتراضية للمدير | **مطلوب للإنتاج** |
| `LIGHTNOVEL_ADMIN_EMAIL` | البريد الإلكتروني الافتراضي للمدير | `admin@novelsparadise.site` |
| `LIGHTNOVEL_CORS_ORIGINS` | النطاقات المسموح لها بالاتصال (CORS) مفصولة بفاصلة | مغلق افتراضياً للإنتاج |
| `LIGHTNOVEL_HOST` | عنوان مضيف السيرفر | `127.0.0.1` |
| `LIGHTNOVEL_PORT` | منفذ تشغيل السيرفر | `5000` |
| `LIGHTNOVEL_DEBUG` | تشغيل وضع المطور والتنقيب | `False` |
| `LIGHTNOVEL_R2_BUCKET` | اسم حاوية Cloudflare R2 | غير محدد (يرفع محلياً) |
| `LIGHTNOVEL_R2_ACCESS_KEY` | مفتاح الوصول لـ R2 | غير محدد |
| `LIGHTNOVEL_R2_SECRET_KEY` | المفتاح السري لـ R2 | غير محدد |
| `LIGHTNOVEL_R2_ENDPOINT` | رابط Endpoint لـ R2 | غير محدد |

## 🚀 طريقة التشغيل (Setup & Run Guide)

### 1. تثبيت الاعتمادات
افتح الطرفية (Terminal) في مجلد `lightnovel-backend` وقم بتثبيت الاعتمادات:
```bash
pip install -r requirements.txt
```

### 2. التشغيل للتطوير المحلي
*   **ويندوز (PowerShell)**:
    ```powershell
    $env:LIGHTNOVEL_SECRET_KEY="ANY_STRONG_RANDOM_SECRET_KEY_HERE"
    $env:LIGHTNOVEL_DEBUG="True"
    python app.py
    ```
*   **لينكس/ماك**:
    ```bash
    export LIGHTNOVEL_SECRET_KEY="ANY_STRONG_RANDOM_SECRET_KEY_HERE"
    export LIGHTNOVEL_DEBUG="True"
    python app.py
    ```

تصفح الرابط مباشرة: **[http://localhost:5000/index](http://localhost:5000/index)**
