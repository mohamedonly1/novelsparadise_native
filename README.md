# Novels Paradise - LightNovel Native (منصة جنة الروايات)

منصة متكاملة لقراءة روايات الويب والخفيفة باللغة العربية، تم تحسينها هندسيًا وتأمينها ضد الثغرات الأمنية الشائعة.

---

## 🛠️ هيكلية المشروع (Architecture)

يتكون المشروع من جزأين رئيسيين:
1. **الواجهة الخلفية (Backend)**: تطبيق Flask متصل بقاعدة بيانات SQLite (`lightnovel.db`) ويقدم واجهة برمجة تطبيقات (RESTful API).
2. **الواجهة الأمامية (Frontend)**: تطبيق صفحة واحدة (SPA) مبني بـ HTML/CSS/JS الأصلي (Vanilla JS)، يدعم التخصيص الكامل لتجربة القراءة والتخزين المحلي.

---

## 🔒 التحسينات الأمنية التي تم إجراؤها (Security Hardening)

تمت معالجة جميع الملاحظات الأمنية الحرجة والحديثة لضمان أمان التطبيق:

1. **حماية صلاحيات Admin/VIP من التزوير**:
   - لا يتم الوثوق بـ `role` المرفق مع التوكن مباشرة كمصدر حقيقة.
   - عند كل طلب حساس، يتم قراءة هوية المستخدم من التوكن ثم جلب صلاحياته ودوره الفعلي ديناميكيًا من قاعدة البيانات.

2. **تأمين حساب المدير الافتراضي ومنع الثوابت غير الآمنة**:
   - تم التخلص تماماً من أي كلمات مرور ثابتة أو افتراضية للمدير في البيئة الإنتاجية.
   - يتوقف التطبيق عن العمل فورًا (Fail-fast) إذا تم تشغيله بوضع الإنتاج دون تحديد متغيرات بيئة مخصصة لحساب المدير (`LIGHTNOVEL_ADMIN_USER` و `LIGHTNOVEL_ADMIN_PASSWORD`). يسمح ببيانات التطوير الافتراضية فقط عند تفعيل وضع التطوير `LIGHTNOVEL_DEBUG=True`.

3. **الحماية من ثغرات XSS المخزنة (Stored XSS)**:
   - تم تطوير محلل ومطهر HTML مدمج ومخصص (`SafeHTMLParser`) يعتمد على مكتبات Python القياسية فقط (دون الحاجة لاعتمادات خارجية).
   - يتم تطهير محتوى الفصول والإعلانات قبل حفظها في قاعدة البيانات للتخلص من وسوم `<script>`، ووسوم الاحتيال، والسمات الخطيرة مثل `onerror` و `onload` وبروتوكولات `javascript:` في الروابط.
   - **الدفاع الوقائي المزدوج (Defense-in-depth)**: بالإضافة إلى التطهير عند الحفظ، يتم تطهير المحتوى ديناميكيًا عند القراءة والتسليم للواجهة الأمامية لمنع ثغرات XSS الموروثة.
   - **الهجرة الأمنية لقاعدة البيانات (Migration)**: يقوم النظام تلقائيًا عند بدء التشغيل بفحص وتطهير كل السجلات القديمة المخزنة مسبقًا في قاعدة البيانات لضمان خلوها من أي أكواد خبيثة موروثة.

4. **تأمين اشتراك VIP**:
   - تم فرض التحقق من صحة معاملة الدفع عبر رموز مشفرة حقيقية. في وضع الإنتاج يمنع استخدام الرموز المحاكية `MOCK_` وتتطلب معاملات دفع حقيقية أو مشفرة ومصادقة لمنع الترقية المجانية غير المشروعة.

5. **فحص صلاحية VIP الفوري**:
   - يتم التحقق من تاريخ انتهاء اشتراك VIP (`vip_expires_at`) ديناميكيًا عند كل طلب للمحتوى المغلق أو جلب الإعلانات، وتخفيض رتبة الحساب تلقائيًا فور انتهاء الاشتراك.

6. **افتراضيات إنتاجية آمنة (Secure Defaults)**:
   - إعدادات المضيف والمنفذ ووضع التنقيب (`debug`) تم ربطها بمتغيرات بيئة مع افتراضية الإغلاق التام لوضع المطورين وعناوين الاستماع الخارجية لمنع فتح التطبيق على كل الشبكات افتراضيًا.
   - يتم إيقاف تشغيل CORS افتراضيًا في الإنتاج ما لم يتم ضبط النطاقات المسموحة عبر `LIGHTNOVEL_CORS_ORIGINS`.

## ⚙️ إعدادات متغيرات البيئة (Environment Variables)

قم بإنشاء ملف `.env` في مجلد `lightnovel-backend` أو قم بتعيين المتغيرات التالية في نظام التشغيل:

| اسم المتغير | الوصف | القيمة الافتراضية |
| :--- | :--- | :--- |
| `LIGHTNOVEL_SECRET_KEY` | المفتاح السري لتشفير التوكنات (JWT) | **لا يوجد افتراضي** (يجب تحديده لمنع توقف التطبيق) |
| `LIGHTNOVEL_DATABASE_FILE` | مسار ملف قاعدة بيانات SQLite | `lightnovel.db` |
| `LIGHTNOVEL_ADMIN_USER` | اسم المستخدم الافتراضي للمدير | **لا يوجد افتراضي** (يتطلب تحديده في الإنتاج) |
| `LIGHTNOVEL_ADMIN_PASSWORD`| كلمة المرور الافتراضية للمدير | **لا يوجد افتراضي** (يتطلب تحديده في الإنتاج لسلامة النظام) |
| `LIGHTNOVEL_ADMIN_EMAIL` | البريد الإلكتروني الافتراضي للمدير | `admin@novelsparadise.site` |
| `LIGHTNOVEL_CORS_ORIGINS` | النطاقات المسموح لها بالاتصال (CORS) | **لا شيء** (مغلق ومحمي افتراضياً للإنتاج) |
| `LIGHTNOVEL_HOST` | عنوان مضيف السيرفر | `127.0.0.1` (محلي آمن ومحمي) |
| `LIGHTNOVEL_PORT` | منفذ تشغيل السيرفر | `5000` |
| `LIGHTNOVEL_DEBUG` | تشغيل وضع المطور والتنقيب | `False` (مغلق ومؤمن افتراضيًا) |

## 🚀 طريقة التشغيل (Setup & Run Guide)

### 1. تشغيل الواجهة الخلفية والواجهة الأمامية معًا
بفضل تحديث نظام المسارات، يقوم سيرفر Flask الآن بخدمة الواجهة الأمامية تلقائيًا عند تصفح الرابط الرئيسي!

1. افتح مبدل الأوامر (Terminal) في مجلد `lightnovel-backend`.
2. قم بتثبيت الاعتمادات اللازمة:
   ```bash
   pip install -r requirements.txt
   ```
3. قم بتشغيل السيرفر بعد تحديد مفتاح أمان تشغيلي محليًا (أو تفعيل وضع المطور للاختبار):
   * في ويندوز (PowerShell):
     ```powershell
     $env:LIGHTNOVEL_SECRET_KEY="ANY_STRONG_RANDOM_SECRET_KEY_HERE"
     $env:LIGHTNOVEL_DEBUG="True"
     python app.py
     ```
   * في لينكس/ماك:
     ```bash
     export LIGHTNOVEL_SECRET_KEY="ANY_STRONG_RANDOM_SECRET_KEY_HERE"
     export LIGHTNOVEL_DEBUG="True"
     python app.py
     ```
4. افتح المتصفح وتصفح الرابط التالي مباشرة:
   **[http://localhost:5000/index](http://localhost:5000/index)**

## 🧪 تشغيل الاختبارات الأمنية (Running Tests)

تم دمج اختبارات آلية للتحقق من أداء التحسينات الأمنية (مثل XSS، وقفل الفصول، والتسجيل/تسجيل الدخول الديناميكي، والتطهير عند الترحيل، والتحقق من صلاحيات التوكنات، وفحص انتهاء VIP):

1. تأكد من وجودك في مجلد `lightnovel-backend`.
2. قم بتشغيل الأمر:
   ```bash
   python test_app.py
   ```
3. ستظهر لك نتيجة الاختبارات كـ `OK` بنجاح تام.
